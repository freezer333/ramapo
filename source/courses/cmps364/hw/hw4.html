Building an application with a Database

For this assignment you will build the database portion of a complete web application written in Node.js and the express framework.  Most of the assignment is limited to writing Javascript that interacts with SQLite, however the last 15 points will require you to build out a new web page in the application as well.

Step 1 - Get the code and start running it!

I have provided you a complete web application, which provides the user interface and calls numerous database functions that you will need to implement.  You can download the zip file of the application at the following link:

https://pages.ramapo.edu/~sfrees/courses/cmps364/hw/hw4.zip

Once you download the file, extract it into a convenient directory and then navigate to it in your command prompt or terminal.  Once inside the directory, install the dependencies:

npm install

Then you can start the application:

npm start

Browse to the application by entering http://localhost:3000 in your web browser.  The application is driven by “mock” data, which will be replaced with your code.

Step 2 - Implement the functions in database.js

Your job is to implement all 13 functions defined in the database.js file.  These functions are being called by the web application, and already have default implementations.  

I recommend you start with the build function.  You must create 3 tables - CUSTOMERS, PRODUCTS, and ORDERS.  You can see the desired schema by examining the customers, products, and orders arrays at the top of database.js - which are “mock” data.  Be sure to delete those arrays as you begin implementing your code however.

The total number of points for each function is shown in database.js, along with complete documentation about what each function should be doing.  Don’t forget to invoke the callback passed in as a parameter to each function - you shouldn’t be returning anything directly!

The purely database code part of the assignment is worth 90 points

Step 3 - Extend the app - Customer Purchase Report

You’ll notice that at the top of the navigation bar of the application there is one button - “Purchase Report” that throws an error when clicked.  I have purposely left his entire part empty.  For the final 10 points of the project, use the structure of the existing application as model for creating a new page.  The purchase report page should look a lot like the Sales Report page, but instead of listing products, it lists customers.  For each customer, display the number of product they’ve purchased, and the total amount of money they’ve spent.  Note that for this report, you will need to create a new route in the reports controller, and a completely new function in database.js.
